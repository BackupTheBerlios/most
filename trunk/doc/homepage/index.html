About

This is the free and open Microcontroller Operating System Tool. It is designed to develope embedded system applications for 16 and 32 bit MCUs without MMU and with at least 32 KB RAM and 64 KB FLASH.
Usually 8bit MCUs do not have enough RAM for multitasking so for smaller Microcontrollers like the 8 bit AVR you may see Arduino. Also a lot of boards with more powerful MCUs with MMU like ARM Cortex A8 and lots of RAM nowadays are extremly cheap und usually come with a real OS like GNU/Linux or Android.

My native language is german, but i decided to write this web site in english even if my english is not good. I hope you can read it.

MOST is not a Hard Real Time OS but it has a short interrupt response time and a fast thread(task) switch. Several libraries and board support packages provide the OS functionality. They are linked with the application to one binary file, this has the advantage that you can debug the hole binary with GDB if the target supports jtag. It is written in C with very less assembly routines. As IDE i use GCC, Eclipse and OpenOCD. Microcontroller boards are ideal for me for development as they are not as complex as personal computers and they provide a rich set of hardware functionality. It also contains a target for an i386 IBM compatible PC, but this can be only used with Qemu (useful for testing code without a target board). It is just a hobby project and work is done in spare time, so it is not suitable for production purpose. But if you like you can help!

If MOST is not suitable for you, next is a list with other small Operating Systems.

History

New version (V-0-1-0-d-7) available:
SAM7-LA2 Board support package: bootloader (y-modem) is working (eth still missing) and frame application available.

New version (V-0-1-0-d-6) available.
Support for new target i386 ibmcompat. PC (only Qemu) and very rudimental! This is taken from the lowlevel wiki (thx to them!)
Support for new target Olimex board SAM7-LA2 (AT91SAM7-A2), porting the sources provided on the olimex web site (for IAR compiler) to the GCC.

21 Januar 2013:

New version (V-0-1-0-d-4) available.
Serial download is now done with Y-Modem, function sections and different build trees for run or debug versions are added, for more information see the doc/CHANGELOG file in the CVS repository.

29 July 2012:

New version (V-0-1-0-d-3) available.
With OpenOCD 0.5.0 and GDB now working within Eclipse. Syslog Client.

11 April 2010:

Development will be continued.
Current state:
Support for ARM7 (AT91SAM7X256) and H800 Microcontrollers is basically working. MOST runs on a Olimex SAM7-EX256 Board and OpenOCD can be used with the Olimex ARM-JTAG or ARM-USB-OCD key for downloading and debugging. The Olimex Board and Jtag Key are available in Germany at Elektronik-Laden or in Ausria at Megaton for about 150 Euro.
